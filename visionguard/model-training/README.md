# Model training

YOLOv8 fine-tuning for VisionGuard. Two datasets, two purposes:

| Source | Classes | Purpose |
|--------|---------|---------|
| **NEU-DET** (public) | 6 surface-defect types | **Headline metric** — precision/accuracy on a named public set, so the numbers are meaningful |
| **Factory Digital Twin** (synthetic) | surface / dimension / color / missing_component | **MLOps loop** — exercises correction → retrain → deploy; matches FactoryOps / EVENT_CONTRACT taxonomy |

> Honest framing (per the portfolio plan): the headline precision/accuracy is the
> NEU-DET number. The synthetic Twin images demonstrate the *pipeline*, not the
> headline metric.

## 1. Get data

**NEU-DET** (manual download — requires accepting terms):
download the NEU surface-defect detection set and unzip so you have
`data/raw/NEU-DET/IMAGES/*.jpg` and `data/raw/NEU-DET/ANNOTATIONS/*.xml`
(Pascal-VOC). Sources: NEU official page or Kaggle `kaustubhdikshit/neu-surface-defect-database`.

**Twin synthetic** (already generated by the Digital Twin):
```bash
cd ../../factory-digital-twin && python synthetic-images/defect_generator.py
```

## 2. Prepare (→ YOLO + data yaml)

```bash
pip install -r requirements.txt
python prepare_dataset.py --source neudet            # -> data/neudet.yaml
python prepare_dataset.py --source twin              # -> data/twin.yaml
```

## 3. Train + export ONNX

```bash
python train.py --data data/neudet.yaml --epochs 50 --name neudet_v1
python train.py --data data/twin.yaml   --epochs 30 --name twin_v1
```
Produces `models/<name>/weights/best.pt` and `models/<name>.onnx`. The inference
server consumes `models/best.onnx` (promote a run via the retraining pipeline's
`deploy_if_better.py`).

## 4. Evaluate (success criteria)

```bash
python evaluate.py --model models/neudet_v1/weights/best.pt --data data/neudet.yaml
```
Checks precision ≥ 90%, latency ≤ 100 ms/image; writes `metrics.json`.

> The original spec lists Jupyter notebooks (01-prepare / 02-train / 03-evaluate).
> These scripts are the runnable, CI-friendly equivalent — wrap them in notebooks
> if you prefer an interactive walk-through.
